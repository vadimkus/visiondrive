# Clinic Group Classes / Events Decision

## Context

The next Altegio adaptation item was group classes and events. For the current solo home-visit product, this is not core. Adding class capacity, attendee rosters, waitlists, and course products would add enterprise-style surface area before there is a real course/workshop workflow.

## Shipped

- Kept group classes/events deferred.
- Added an EN/RU Knowledge Base guide for handling occasional workshops manually.
- Updated backlog, architecture, data model, chunks, and docs index.

## Recommended Manual Workflow

For occasional workshops:

- create blocked time for the event,
- record event-related costs in Finance,
- track interested clients in patient notes, CRM notes, or simple tags,
- collect payment as a normal patient payment only when there is a real patient record.

Do not add group booking tables until workshops become repeatable revenue.

## Deferred Until Needed

- Course/service products with capacity.
- Attendee rosters.
- Waitlists.
- Group payment allocation.
- Public booking for classes.
